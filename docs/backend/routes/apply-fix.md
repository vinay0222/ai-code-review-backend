# POST /apply-fix

Generates AI-powered code fixes for a PR and opens a new pull request with the patched files.

**Auth:** `requireAuth` — user must be logged in with a connected GitHub account.

---

## Request Body

```json
{
  "pr_url":     "https://github.com/owner/repo/pull/42",
  "project_id": "firestore-doc-id"
}
```

---

## Response

### Success
```json
{
  "success":         true,
  "pr_url":          "https://github.com/owner/repo/pull/43",
  "pr_number":       43,
  "branch":          "ai-fix-42-1712345678901",
  "overall_summary": "Fixed null check on LoginForm and removed dead variable in utils.js",
  "files_fixed":     2,
  "patches": [
    {
      "path":    "src/auth/LoginForm.jsx",
      "diff":    "--- original\n+++ ai-fixed\n@@ ... @@\n-  if (x)\n+  if (x != null)",
      "changes": ["Fixed null check on line 42", "Removed unused import `useRef`"]
    }
  ],
  "request_id": "uuid-v4"
}
```

### No Changes
```json
{
  "success":         false,
  "message":         "AI found no changes to apply...",
  "overall_summary": "",
  "patches":         [],
  "request_id":      "uuid-v4"
}
```

### GitHub Operation Failed (patches still returned)
```json
{
  "error":           "GitHub operation failed: Resource not accessible by integration",
  "patches":         [...],
  "overall_summary": "...",
  "request_id":      "uuid-v4"
}
```

---

## Processing Steps

### 1. Parse PR URL + fetch PR metadata
- `parsePrUrl(pr_url)` → `{ owner, repo, pull_number }`
- GitHub API: `GET /repos/{owner}/{repo}/pulls/{pull_number}`
- Extracts: `baseBranch` (e.g. `main`), `headSha` (latest commit SHA)
- **Guards:** PR must be open. Closed PRs are rejected with 400.

### 2. Fetch changed files list
- GitHub API: `GET /repos/{owner}/{repo}/pulls/{pull_number}/files`
- Filters: skip `status === 'removed'`, binary extensions, files with 0 changes
- Cap: max 6 files processed

#### Skipped extensions
```
.png .jpg .jpeg .gif .webp .svg .ico .pdf
.woff .woff2 .ttf .eot .otf .bin .lock .map
.min.js .min.css
```

### 3. Fetch file contents
- GitHub API: `GET /repos/{owner}/{repo}/contents/{path}?ref={headSha}`
- Response is base64-encoded content — decoded with `Buffer.from(..., 'base64')`
- Skip files larger than **30 000 bytes**
- Cap content sent to OpenAI at **8 000 chars per file**

### 4. Load context from Firestore (optional)
If `project_id` is provided:
- Find the most recent review for this exact `pr_url` → use its `issues[]` as fix context
- Load `project.rules` for the fix prompt

### 5. OpenAI fix generation
```js
openai.chat.completions.create({
  model: 'gpt-4o',
  messages: [
    { role: 'system', content: buildFixSystemPrompt() },
    { role: 'user',   content: buildFixUserPrompt(files, issues, rules, prDetails) },
  ],
  response_format: { type: 'json_object' },
  temperature: 0.1,        // low temp for deterministic fixes
})
```

**System prompt** instructs the model to:
- Fix ONLY the identified issues
- Never rewrite unrelated code
- Preserve all formatting, indentation, and comments
- Only include files where changes were made

**Expected model response:**
```json
{
  "fixes": [
    {
      "path":          "src/auth/LoginForm.jsx",
      "fixed_content": "<full corrected file content>",
      "changes":       ["Fixed null check on line 42"]
    }
  ],
  "overall_summary": "Fixed 1 issue in LoginForm.jsx"
}
```

### 6. Generate unified diffs
Uses `diff` package v5 (`require('diff')`):
```js
Diff.createPatch(path, originalContent, fixedContent, 'original', 'ai-fixed')
```

> **Why `diff@5`?** `diff@8+` is ESM-only and breaks `require()`. Always keep `"diff": "^5.x"` in `package.json`.

### 7. GitHub Git Tree API — create branch + commit

```
GET  /git/ref/heads/{baseBranch}        → baseSha
GET  /git/commits/{baseSha}             → baseTreeSha
POST /git/blobs × N files               → blobSha[] per file
POST /git/trees { base_tree, tree[] }   → newTreeSha
POST /git/commits { message, tree, parents: [baseSha] }  → newCommitSha
POST /git/refs { ref: 'refs/heads/ai-fix-...', sha }     → branch created
```

Branch name format: `ai-fix-{pull_number}-{Date.now()}`

**Commit message:**
```
🤖 AI Patch Fix for PR #{pull_number}

{overall_summary}

Files changed ({count}):
  - {path}
  ...

Generated by AI Code Review | Original PR: {pr_url}
```

### 8. Create PR
```
POST /repos/{owner}/{repo}/pulls
{
  title: "🤖 AI Patch Fix for #{pr_number} — {pr_title}",
  body:  "## 🤖 AI Patch Fix\n> ...",
  head:  "ai-fix-{n}-{ts}",
  base:  "{baseBranch}"
}
```

The new PR targets the **original PR's base branch**, not the feature branch itself.

---

## Error Handling

| Scenario | HTTP Status | Notes |
|---|---|---|
| No GitHub token | 401 | User must connect GitHub |
| PR not found / closed | 400 / 502 | Clear error message |
| No eligible files | 400 | All files binary/deleted/too large |
| OpenAI fails | 500 | |
| GitHub create branch/PR fails | 502 | **Patches still returned** so user can see diffs |

---

## Constants

| Name | Value | Description |
|---|---|---|
| `MAX_FILE_BYTES` | 30 000 | Max file size to process |
| `MAX_FILES` | 6 | Max files per fix request |
| `CONTENT_BYTES_PER_FILE` | 8 000 | Chars sent to OpenAI per file |
